# /fusion-benchmark-report — Benchmark Report Generator

Reads solo and fusion results from `benchmark/results/` and generates
`benchmark/results/REPORT.md` with per-task grading, blind-spot analysis,
and an honest verdict.

## Steps

### 1. Gather data

```bash
ls benchmark/results/solo-*.md benchmark/results/fusion-*.md 2>/dev/null
```

If no results found, tell the user to run `/fusion-benchmark` first.

### 2. Read the rubric

Read `benchmark/grade.md` for the scoring dimensions and weights:

| Dimension    | Weight | Multiplier |
|--------------|:------:|:----------:|
| Correctness  | 40     | ×4         |
| Completeness | 25     | ×2.5       |
| Blind spots  | 20     | ×2         |
| Code quality | 15     | ×1.5       |

Each dimension rated 0–10. Formula:
`score = (correctness × 4) + (completeness × 2.5) + (blind_spots × 2) + (code_quality × 1.5)`

### 3. Grade each task

For every task that has BOTH a solo and fusion result:

1. Read `benchmark/results/solo-XX.md` — grade it on all four dimensions.
2. Read `benchmark/results/fusion-XX.md` — grade it on all four dimensions.
3. Calculate weighted scores for both.
4. Compute `Δ = fusion − solo`.

Grade **blind** — evaluate each answer on its own merits without bias toward
which mode produced it. The fusion answer will naturally be longer (it includes
panelist responses + judgment); grade only the **Final Synthesized Answer**
section, not the panelist raw responses.

### 4. Generate REPORT.md

Write `benchmark/results/REPORT.md` using this template:

> **Orchestrator/Judge:** <model that ran the benchmark>
> **Panelists:** <list of panelist model names>

> **Tasks Run:** <count>

---

### 5. Display

Show the complete REPORT.md to the user. Also show the file path so they
can find it easily.

## Rules

- Grade the **synthesized answer**, not the raw panelist responses.
- Be honest. If solo won, say so. If fusion didn't help, say so.
- The blind-spot analysis is the most important section — spend time on it.
- If a task is missing one arm (solo or fusion), mark it N/A and explain why.
